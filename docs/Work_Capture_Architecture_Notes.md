# Work Capture 構造設計 検討メモ

> **思考を止めずに、仕事を前へ進める。**

## 目的

Work Capture は ToDo
アプリではなく、「思考を構造化し、仕事へ変換するシステム」を目指す。

------------------------------------------------------------------------

# システム構成（想定）

## フロントエンド

-   Next.js (App Router)
-   shadcn/ui
-   Tailwind CSS

## ホスティング

-   Vercel

## データベース

-   Neon (PostgreSQL)

## AI

-   LLM API（構造化担当）

## 音声

-   スマホで録音
-   音声文字起こし
-   AIへ渡して構造化

------------------------------------------------------------------------

# 全体フロー

``` text
SPで録音
↓
音声アップロード
↓
文字起こし
↓
AI構造化
↓
確認画面
↓
Neonへ保存
↓
PCでInbox Zero
↓
Dashboardで実行
```

------------------------------------------------------------------------

# スマホの役割

Capture

-   起動したら録音画面
-   思考を止めない
-   入力は3秒
-   テキスト入力は補助

------------------------------------------------------------------------

# PCの役割

## Inbox Zero

-   AI解析確認
-   担当決定
-   優先順位
-   Project
-   Context
-   保存

仕事を作る場所。

## Dashboard

-   今日やること
-   アクション実行
-   完了

仕事を終わらせる場所。

------------------------------------------------------------------------

# AIの役割

AIは「要約」ではなく「構造化」。

抽出する候補

-   目的
-   背景
-   タスク
-   期限
-   アクションプラン
-   メモ
-   判断事項
-   次の一歩
-   Project候補
-   Context候補

AIは候補を提示し、人間が確定する。

------------------------------------------------------------------------

# DBへ保存する方向性

親データ

-   Work Capture

子データ

-   Tasks
-   Action Items
-   Notes
-   Decisions

加えて

-   元音声
-   文字起こし
-   AI解析結果
-   人間が確定した内容

------------------------------------------------------------------------

# 音声文字起こし案

## 案A

Web Speech API

メリット - 軽い - リアルタイム

デメリット - ブラウザ依存

## 案B（採用候補）

MediaRecorder ↓ Next.js API ↓ Speech-to-Text ↓ LLM ↓ Neon

こちらを第一候補とする。

# 開発方針

完成してから使うのではなく、

**毎日使いながら育てる。**

1つのPhaseが完成したら実際の仕事で試し、 改善を繰り返す。

------------------------------------------------------------------------

# 判断基準

新機能を追加する時は必ず問う。

> この機能は「迷う時間」を減らすか？

YESなら採用候補。

NOなら採用しない。

------------------------------------------------------------------------

# 現時点での設計思想

-   SP = 思考を預ける場所
-   PC = 仕事を前へ進める場所
-   AI = 思考を構造化する編集者
-   人間 = 判断し、実行する責任者

Work Capture は「タスク管理ツール」ではなく、

**思考を仕事へ変換するOS** を目指す。
