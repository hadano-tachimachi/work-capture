# Work Capture UI設計メモ

> **思考を止めずに、仕事を前へ進める。**

---

# 1. UI全体の考え方

Work Capture は、スマートフォンとPCで役割を分ける。

## スマートフォン

**Capture**

思いついたことを、最短で預ける場所。

- 起動したら録音画面
- 音声入力が主役
- テキスト入力は声を出せない時の補助
- タスク確認・編集・完了への動線は残す
- ただし初手画面では主張させない

---

## PC

**Organize / Execute**

AIが構造化した内容を確認し、仕事として前へ進める場所。

PC側は2つのモードに分ける。

1. **Inbox Zeroモード**
   - 未整理のWork Captureを処理する
   - AI解析結果を確認し、仕事として確定する

2. **Dashboardモード**
   - 今日やる仕事を進める
   - タスクを実行・完了する

---

## Inbox Zeroモードの役割

この画面でやることは以下。

- 元の音声を確認する
- AIの分解結果を見る
- タスク化するか判断する
- 担当者を決める
- 優先度を決める
- プロジェクトを決める
- コンテキストを決める
- 保存して次へ進む

---

## Inbox Zeroモードの思想

```text
未整理の思考
↓
AIが構造化
↓
人間が確認・判断
↓
仕事として確定
```

この画面は「仕事をする場所」ではない。

**仕事を作る場所。**

# 5. 2ペインか3ペインか

## Inbox Zeroモード

基本は2ペインでよい。

```text
┌──────────────────────┬────────────────────────────┐
│ 元の入力              │ AI解析結果＋確定操作         │
└──────────────────────┴────────────────────────────┘
```

ただし、大画面では3ペインも検討できる。

```text
┌──────────┬────────────────────┬────────────────────┐
│ 一覧      │ AI解析プレビュー     │ 編集・確定           │
└──────────┴────────────────────┴────────────────────┘
```

MVPでは2ペインから始める方がシンプル。

---

## Dashboardモード

3ペインが合う。

```text
┌──────────────┬────────────────────┬────────────────────┐
│ 今日やること  │ 今日の予定          │ 詳細・実行           │
└──────────────┴────────────────────┴────────────────────┘
```

理由は、実行時には

- 優先順位
- 時間
- 詳細

を同時に見たいから。

# 6. UI設計の判断基準

新しい画面や機能を追加する時は、必ず問う。

> **このUIは迷う時間を減らすか？**

YESなら採用候補。

NOなら削る。

# 7. 現時点のUI方針まとめ

## SP

- 初手は録音画面
- 音声入力が主役
- テキスト入力はボトムシート
- メニューは控えめ
- 確認画面ではAIの分解結果を見せる

## PC

- Inbox ZeroモードとDashboardモードを分ける
- Inbox Zeroは「整理」
- Dashboardは「実行」
- 朝はInbox Zero
- 日中はDashboard
- 2つのモードを混ぜない


# 9. UIの本質

スマホは、思考を止めないための入口。

PCは、仕事を前へ進めるための司令室。

Work Capture のUIは、
「記録するための画面」ではなく、
**迷わず仕事を進めるための画面**である。
