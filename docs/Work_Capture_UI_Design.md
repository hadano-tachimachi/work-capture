# Work Capture UI設計メモ

> **思考を止めずに、仕事を前へ進める。**

---

# 1. UI全体の考え方

Work Capture は、スマートフォンとPCで役割を分ける。

## スマートフォン

**Capture**

思いついたことを、最短で預ける場所。

- 起動したら録音画面
- 音声入力が主役
- テキスト入力は声を出せない時の補助
- タスク確認・編集・完了への動線は残す
- ただし初手画面では主張させない

---

## PC

**Organize / Execute**

AIが構造化した内容を確認し、仕事として前へ進める場所。

PC側は2つのモードに分ける。

1. **Inbox Zeroモード**
   - 未整理のWork Captureを処理する
   - AI解析結果を確認し、仕事として確定する

2. **Dashboardモード**
   - 今日やる仕事を進める
   - タスクを実行・完了する

---

## 1-1. 製品の姿と道のり

最終的には、TODO ツールに近い画面も増えていく。

ただし Work Capture の本質は「リストを管理すること」ではない。

```text
Capture（預ける）
  ↓
構造化（AI が候補を作る）
  ↓
Organize（人間が確定する）
  ↓
Execute（実行・完了する）
```

タスク一覧や完了ボタンは、この流れの**最後**に置く。  
いきなり TODO アプリから始めない。底に至るまでの順番を崩さない。

その道のり自体が、このアプリケーションの設計である。

## Inbox Zeroモードの役割

この画面でやることは以下。

- 元の音声を確認する

- タスク化するか判断する
- 担当者を決める
- 優先度を決める
- プロジェクトを決める
- コンテキストを決める
- 保存して次へ進む

## Inbox Zeroモードの思想

```text

↓

```

この画面は「仕事をする場所」ではない。

# 3-2. Dashboardモード（Phase 7）

## 目的

Inbox Zero で確定した `tasks` を、実行・完了する。

整理は Inbox Zero で終わっている。この画面では判断しない。選んで、終わらせる。

---

## MVP：2 ペイン（実装対象）

Inbox Zero と同型。左に一覧、右に詳細。

```text
┌──────────────────────────────────────────────────────────────────────────┐
│ Work Capture              [ Inbox Zero (2) ]  [ Dashboard ● (5) ]       │
├─────────────────────────────┬────────────────────────────────────────────┤
│ 今日やること                 │  山田さん見積作成                           │
│ 未完了 5 件                  │  ─────────────────────────────────────    │
│─────────────────────────────│  高  ·  今日  ·  見積案件  ·  自分          │
│ ● 山田さん見積作成      高   │                                            │
│   期限: 今日                 │  説明                                      │
│                              │  山田さんの見積を作成し共有する              │
│ ○ HP修正                中   │                                            │
│   期限: 7/3                  │  次の一歩                                  │
│                              │  見積項目を入力する                         │
│ ○ 撮影準備              低   │                                            │
│                              │  メモ（折りたたみ）                         │
│ ── 保留 ──                   │  粗利は知らせた方が良い                     │
│ ○ SNS投稿（保留）       中   │                                            │
│                              │  ┌────────────────────────────────────┐  │
│                              │  │    [ 完了 ]        [ 保留 ]         │  │
│                              │  └────────────────────────────────────┘  │
└─────────────────────────────┴────────────────────────────────────────────┘
```

### 左ペイン：一覧

| 項目 | 内容 |
|------|------|
| 対象 | `status` が `todo` または `on_hold` のタスク |
| 並び | 優先度（高→低）→ 期限 → 作成日 |
| 1 行 | タイトル / 優先度 / 期限 / プロジェクト（あれば） |
| 選択 | 紫左ボーダー（Inbox カードと同じ） |

MVP では「今日」で厳密フィルタしない。Inbox を空にした直後は、確定タスクがすべて候補になる。

### 右ペイン：詳細

| ブロック | データソース |
|---------|-------------|
| タイトル・メタ | `tasks` テーブル |
| 説明 | `tasks.description` |
| 次の一歩 | `structured_items`（`work_capture_id` で JOIN） |
| メモ | `structured_items`（折りたたみ） |

### 操作

| ボタン | 動作 |
|--------|------|
| 完了 | `status` → `done`、一覧から消え、次を自動選択 |
| 保留 | `status` → `on_hold`、保留セクションへ |

編集・並び替え・アクションプランのチェックは Phase 7 後半。

### URL

- `/dashboard` — PC のみ（`md:` 以上）
- ヘッダーで Inbox Zero と切替

---

## 将来像：3 ペイン（参考）

```text
└──────────────┴────────────────────┴────────────────────┘
```

2 ペイン MVP では、右ペイン上部の期限表示と「次の一歩」1 行で代替する。

スマホは、思考を止めないための入口。
